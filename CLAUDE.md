## 실행한 작업 md 관리 (`docs/history/`)

개발은 **작업 내용을 md로 작성 → 그 md를 Claude Code로 실행 → 다음 작업 md 작성 → 실행** 패턴으로 진행한다.
`docs/history/`에는 **실제로 실행한 그 md(작업 지시서/계획서)** 를 보관한다. 3명이 각자 작업하므로, 이 md들을 통해 서로 무엇을 실행했는지 싱크한다.

### 작업 전 — 싱크

- 작업 시작 전 `docs/history/` 하위의 최근 md들을 읽어, 다른 사람이 무엇을 실행했고 현재 어디까지 진행됐는지 파악한 뒤 진행한다.
- 파일명이 날짜순으로 정렬되니 최신 파일부터 확인한다.

### 실행한 md 보관 규칙

- 실행한 작업 md 1개 = 파일 1개로 `docs/history/`에 보관한다.
- **파일명**: `YYYY-MM-DD-<git아이디>-<작업슬러그>.md`
  - 예: `2026-06-05-inkangdev-주가적재배치.md`
  - `<git아이디>`는 `git config user.name` 값을 사용한다.
