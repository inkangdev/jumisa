# Jumisa 작업 슬러그 (v1.1 기준)

> 기준 문서: `docs/기능정의서_v1.1.md`
> 각 슬러그는 `docs/history/` 작업 지시서 파일명(`YYYY-MM-DD-<git아이디>-<슬러그>.md`)에 그대로 사용한다.

## 완료

- `초기셋업` · `로그인화면` · `시세적재배치` · `회원인증`

## A. 저평가 스크리너 (1순위 — 서비스 코어)

| 슬러그 | 내용 | 선행 |
|---|---|---|
| `재무지표적재배치` | 금융위 API → `stock_financials` (PER/PBR/EV·EBITDA·성장률 원천) | — |
| `일봉적재배치` | 마감 후 종가+가치지표 → `stock_daily` | — |
| `저평가점수산출` | 공식 확정: PER30+PBR30+EV25+성장15 → `undervalue_score` | 재무·일봉 |
| `스크리너API` | 섹터/업종 필터, PER·PBR·EV 상한, 점수·PER·PBR 정렬 | 점수산출 |
| `스크리너화면` | 슬라이더 필터, 쉬운 용어 병행, 점수 뱃지, ☆토글, 종목→거래 | 스크리너API |

## B. 미국 주식 (확정 — A·C와 병행)

| 슬러그 | 내용 |
|---|---|
| `미국종목마스터` | 미국 종목 마스터 적재 (`stock` 확장 또는 분리) |
| `FMP시세적재` | FMP API → 미국 종목 시세 적재 (국내 배치와 동일 구조) |
| `FMP재무적재` | FMP API → 미국 종목 재무지표 (점수 공식 동일 적용) |
| `환율연동` | 체결 시점 실시간 환율 — 매매·대결·평가액 공통 사용 |

## C. 모의투자 솔로 (2순위)

| 슬러그 | 내용 |
|---|---|
| `포트폴리오모델` | 포인트·보유종목(평단/수량)·거래내역 스키마 |
| `매매API` | 시장가 즉시 체결, 1시간 적재가 기준, 환율 적용, 평단 재계산 |

| `초기포인트지급` | 가입 즉시 100만P (회원인증에 연결) |

## 권장 진행 순서

- 메인 라인: **A(스크리너) → C(솔로 매매) → D(대결)**
- B(미국)는 A·C와 병행하되, `환율연동`은 C의 `매매API` 전에 완료해야 함
- E·F·G는 코어 안정화 후

## 슬러그 진입 전 결정 필요 (미결)

- `소셜로그인` — 카카오/네이버 연동 범위·우선순위
- 포인트 재충전 정책 — 충전 한도·주기
- `대결` 시세 갱신 주기 — 1시간 적재가 vs 실시간
- `관심종목알림` / `랭킹히스토리` / `뉴스화면설계` — 시안 미정의, 화면 설계 선행
