# Jumisa 기능 정의서 v1.4

> 기준 시안: `docs/design/remixed-13412f58.tsx`
> v1.4는 v1.3 전체를 유지하고, **모의투자 대결(5장)** 에 **대결 봇(컴퓨터 참가자) 자동 매매** 요건을 추가한다(5.6). 사람이 적어도(예: 3인) 방을 채우고, 자동으로 매매하는 컴퓨터 상대와 겨룰 수 있게 한다.

## 0. 화면 구성 (하단 네비게이션)

시안 기준 4개 탭: **스크리너 / 대결 / 관심종목 / 더보기**

## 1. 저평가 주식 스크리너

- 국내/미국 탭 구분 (**미국 포함 확정**)
- 섹터/업종 필터, PER·PBR·EV/EBITDA 상한 슬라이더 필터
- 정렬: 점수순 / PER순 / PBR순
- 저평가 점수 산출 및 랭킹 (점수·색상 뱃지 표시)
- PER/PBR/EV/EBITDA 등 주식 용어 → 입문자용 쉬운 용어 병행 표기
  (예: PER "주가÷순이익", PBR "주가÷순자산", EV/EBITDA "기업가치÷영업이익")
- 종목 → 관심종목 ☆ 토글, 종목 클릭 시 거래 화면 진입

### 저평가 점수 공식 (확정)

`점수 = PER 30% + PBR 30% + EV/EBITDA 25% + 성장률 15%`

### 데이터 소스

- 국내 주가 (1시간 단위) → 한국투자증권 Open API
- 국내 재무지표 → 금융위원회 공공데이터포털
- 미국 주식 (시세·재무) → FMP API

### 뉴스 / AI 분석

- 네이버 금융 RSS로 종목별 뉴스 수집
- Claude API로 뉴스 감성 분석 및 종목 현황 요약 제공
- 미국 종목 뉴스 → FMP 뉴스 엔드포인트
- ※ 시안에 화면 미정의 — 화면 설계부터 필요

## 2. 로그인 / 회원

### 로그인

- 아이디(닉네임)/비밀번호 + 간편 소셜 로그인 (카카오 / 네이버)
- 가입 시 아바타(이모지) 선택

### 더보기 (회원 정보)

- 포인트 충전, 로그아웃
- 공지사항 / FAQ
- 모의투자 포인트 / 보유 종목 / 수익률 통계
- 보유 주식 관리 (수동 입력, 종목 코드 입력 시 현재가 자동 연동)
- **대결 전적 / 보유 메달**: 누적 우승 횟수·메달(금/은/동) 개수 표시 → 5.5 참조

## 3. 공통

### 포인트

### 매수 / 매도

## 5. 모의투자 대결 (멀티플레이)

> 시안의 핵심 기능. 같은 시작 포인트로 출발해 기간 내 수익률로 순위를 겨룬다.

### 5.1 요건

- **대결 로비**: 참가 가능한 방 목록 / 진행 중인 대결 목록, 새 대결방 만들기
- **방 생성**: 방 이름, 대결 기간(3 / 7 / 14 / 30일), 시작 포인트(50만 / 100만 / 300만P), 최대 인원(2~20명), 거래 가능 시장(국내+미국 / 국내 / 미국)
- **대기방**: 초대 코드 공유, 참가자 목록/준비 상태, 방장이 대결 시작. 시작 시 모든 참가자에게 시작 포인트 동시 지급, 시작 후 추가 충전 불가
- **대결 진행**: 내 순위 카드(순위/수익률/총자산)·D-day, 순위 탭(랭킹+수익률 막대그래프), 내 투자 탭(현금/주식평가액/보유종목 손익), 거래 탭(시장 필터 국내/미국)
- **순위 기준**: 종료 시점 **총자산(현금 + 주식평가액)** 의 수익률

### 5.2 1차 구현 현황 (인지)

방 생성·입장·시작·매수/매도·순위·포트폴리오까지 동작하는 1차 슬라이스가 머지됨(국내 종목, 1시간 적재가 기준). **단, 거래 시스템으로서의 정확성·공정성·완결성 요건이 미비하여 아래 보완이 필요하다.**

### 5.3 보완 필요 항목 (우선순위별)

> 거래 시스템의 4대 기둥 = **정합성 / 가격 / 정산 / 회계** + 인가·성능·구조.

**P0 — 정합성·안전**

- **동시 거래 안전성**: 한 참가자의 현금·보유 수량이 동시 요청에 덮어써지지 않도록(잔고 초과 매수·음수 수량 방지).
- **거래 멱등성**: 더블클릭·재전송 중복 체결 방지.
- **대결 종료 강제**: 종료 시각 이후 거래 차단(현재 마감 후에도 무기한 거래 가능).
- **결과 확정(정산)**: 종료 시 최종 순위·총자산을 고정(freeze)하고 '종료'로 전환. 종료 후 시세 변동이 결과·우승자를 바꾸지 않도록. (로컬 배치 스케줄러 의존 시 PC 꺼지면 정산 누락 → **조회 시점 정산** 방식 고려)
- **조회 권한(인가)**: 비참가자가 임의 방의 상세·**초대코드**를 보지 못하도록.
- **시세 부재 처리**: 현재가가 없을 때 평가액을 평단으로 조용히 대체하지 말 것(수익이 0%로 숨겨져 순위 왜곡). 명시적으로 "가격 없음" 표시.
- **초대코드 충돌 처리**: 중복 생성 시 재시도.

**P1 — 거래 현실성·회계**

- **체결가 기준 확정(선결)**: "1시간 적재가"는 한 시간 동안 가격이 안 움직여 장중 변동·공정성이 없음. 실시간/더 잦은 적재/체결 시점 조회 중 택1.
- **시세 신뢰성**: 시세 수집이 멈추면 거래·평가 불가/왜곡. 라이브 기능이므로 시세 공급 신뢰성 확보.
- **종목별 최신가 캐시**: 평가·순위 때마다 시세 이력을 반복 조회하지 않도록.
- **거래 원장 보강**: 거래마다 수수료/세금·체결 후 잔고·**실현손익** 기록 → 대사 가능, 실현손익 표시.
- **금액 정밀도**: 평단·평가의 정수 절삭 오차 제거.
- **시작 조건**: 최소 인원·준비 상태 충족 시에만 시작(현재 방장 혼자 시작 가능, 준비 상태는 표시만).

**P2 — 스펙 완성·확장**

- **거래시장 설정 실효화**: 현재 kr/us/both 설정이 거래에 반영되지 않음. 미국 종목·환율 없이 'us'/'both' 방은 거래 불가.
- **미국 종목 + 환율**: 미국 시세·재무 + 체결 시점 실시간 환율을 매매·평가액에 적용.
- **결과 영속화**: 종료된 대결의 최종 순위·실현손익·우승자 보관.
- **순위·평가 성능**: 참가자·보유 수에 비례하는 반복 조회를 단일 집계로. 거래 종목 일부만 노출되는 문제 정리.
- **거래 코어 공용화(구조)**: 솔로 모의투자(4장)와 대결(5장)이 가격 조회·체결·평단·현금·환율·회계 로직을 공유하도록 공용화 → 중복·표류 방지.

**P3 — 부가**

- 거래 화면 실시간 가격 갱신 / 진행 상태 영속(새로고침 대응) / 가격 알림·랭킹 히스토리 / **대결 봇 → 5.6 으로 정식화(v1.4)**.

### 5.4 선결 결정 (코드 전 합의)

- 대결 시세 갱신 주기 (실시간 vs 적재가)
- 시세 수집 신뢰성 방식
- 수수료·거래세 적용 여부

### 5.5 메달 부여 & 명예의 전당(랭킹)

> 대결이 끝나면 최종 순위에 따라 **메달**을 부여하고, 메달은 **사용자 정보와 연동**되어 누적된다. 어떤 사용자가 어떤 메달을 몇 개 보유했는지를 **전체 랭킹(명예의 전당)** 화면으로 보여준다. 대결을 단발성 게임이 아니라 **누적 명예 시스템**으로 확장하는 것이 목적이다.

#### 5.5.1 메달 부여 규칙

- 대결이 **종료(정산 확정, 5.3 '결과 확정')** 되는 시점에 최종 순위가 고정되며, 그 순위를 기준으로 메달을 부여한다.
  - **1등 → 🥇 금메달**, **2등 → 🥈 은메달**, **3등 → 🥉 동메달**.
  - 4등 이하는 메달 없음. (순위만 전적에 기록)
- 메달 부여는 **정산 시점에 1회만** 확정한다. 정산 후 시세 변동·재조회로 메달이 번복되지 않는다(5.3 freeze 원칙과 일치).
- **부여 조건(공정성)**: 메달이 의미를 가지려면 최소 참가 인원이 필요하다. **최소 인원 미만(예: 2인 미만, 정확 기준은 미결)으로 끝난 방은 메달 미부여** 또는 별도 처리. **봇은 이 '인원'에 포함하지 않는 것을 권장**(5.6.3 연동). → 미결.
- **동순위(타이) 처리**: 총자산 수익률이 동일해 순위가 겹칠 때의 메달 분배 규칙 미확정. → 미결.
- 부여된 메달은 **회원(member)** 에 귀속되어 누적되며, 대결방·시점 정보와 함께 기록되어 "언제 어떤 대결에서 받았는지" 추적 가능해야 한다.

#### 5.5.2 사용자 ↔ 메달 연동 (보유)

- 회원별로 **보유 메달 집계**(금/은/동 각 개수)와 **수상 이력**(대결방, 등수, 일시)을 보유한다.
- 회원 프로필(더보기, 2장)에 **내 메달 요약**(🥇 n · 🥈 n · 🥉 n)과 누적 대결 전적(참가/우승 횟수)을 노출한다.
- 메달은 회원 식별자에 종속되므로, **회원 탈퇴 시 메달·랭킹 데이터 처리 정책**(삭제 vs 익명 보존)이 필요하다. → 미결.

#### 5.5.3 랭킹(명예의 전당) 화면

- **전체 사용자 메달 랭킹** 화면을 신설한다. 어떤 사용자가 어떤 메달을 몇 개 보유했는지로 순위를 매겨 보여준다.
- **랭킹 정렬 기준(확정 필요)**: 1차안 — **금 > 은 > 동 순 가중**(예: 금 3 / 은 2 / 동 1점)으로 합산 점수 내림차순, 동점 시 금메달 수 → 은 → 동 순 비교. → 정렬·가중치 미결.
- **행 표시**: 순위, 사용자 아바타(이모지)·닉네임, 🥇/🥈/🥉 개수, (선택) 누적 점수·참가/우승 횟수.
- **본인 하이라이트**: 목록에서 로그인 사용자 본인 행을 강조하고, 목록 밖이면 내 순위를 별도 표기.
- **진입 동선**: 대결 탭 또는 더보기에서 진입(정확한 위치 미결). 대결 **종료 결과 화면**에서 "명예의 전당으로" 이동 동선 제공.
- **범위**: 전체 누적 기준(전역 랭킹). 기간별(이번 주/이번 달)·방별 랭킹은 후속 확장 옵션. → 미결.

#### 5.5.4 백엔드/데이터 (방향, 상세는 작업 슬러그에서)

- 메달 부여는 **결과 확정(정산) 트랜잭션 안에서** 함께 처리해 정산-메달 정합성을 보장한다(메달만 누락/중복되지 않도록).
- 수상 이력 저장 + 회원별 메달 집계 조회가 필요하다(집계는 매 조회 산출 vs 카운터 캐시 중 택1, 5.3 성능 원칙 따름). → 구현 방식 미결.
- 멱등성: 같은 대결의 정산이 재시도돼도 메달이 중복 부여되지 않아야 한다(대결당 메달 1세트).

### 5.6 대결 봇 (컴퓨터 참가자) — v1.4 신규

> 사람만으로 방을 채우기 어려운 상황(예: 늘 3명)에서도 대결을 풍성하게 하기 위해, **자동으로 매매하는 컴퓨터 참가자("봇")** 를 방에 추가할 수 있게 한다. 봇은 사람과 **동일한 거래 규칙·동일한 시세** 로만 움직이는 또 한 명의 참가자이며, 새로운 매매 엔진이 아니라 **기존 거래 로직을 스케줄이 대신 실행**하는 형태로 구현한다.

#### 5.6.1 봇 편성 (방 생성 / 대기방)

- 방 생성 시 **봇을 N명 추가**할 수 있다. 봇 합계는 방의 **최대 인원(maxPlayers) 한도를 사람과 공유**한다.
- 봇마다 **성격(전략)** 과 **난이도** 를 선택한다(미선택 시 기본값).
- 봇은 사람 참가자와 동일하게 입장하며, 대결 시작 시 **같은 시작 포인트**를 받는다(별도 특혜 없음).
- 대기방/참가자 목록·순위·포트폴리오에서 봇은 **🤖 아이콘과 캐릭터 이름**으로 사람과 구분 표시한다.

#### 5.6.2 봇 자동 매매 (행동)

- **공정성 원칙**: 봇은 사람과 **같은 시세(현재 1시간 적재가)** 만 보고 판단하며, 미래 정보·실시간 우위·잔고/보유 검증 면제 같은 특혜가 없다. 매수 가능 잔고·보유 수량 검증을 사람과 동일하게 거친다.
- **매매 시점**: 사람과 동일한 가격 기준에 맞춰 **주기적으로 자동 매매**한다(시세 갱신 시점마다 1회). 같은 시세를 두 번 거래하지 않는다.
- **성격(전략) — 예시(확장 가능)**:
  - **가치형**: 저평가 점수/저PER·저PBR 종목을 매수해 **장기 보유**, 매도 드묾.
  - **모멘텀형**: 직전 구간 **급등 종목 추격**, 하락 시 손절·급등 시 익절(회전 빠름).
  - **랜덤형**: 무작위 매매. 코믹 요소이자 **벤치마크**(맞히기 쉬운 상대), 배선 검증용.
- **난이도**: 전략의 정확도·과감성(매수 비중·손절/익절 기준·무작위 비율)으로 **하/중/상** 조절.
- **거래 반영**: 봇의 매수/매도도 사람과 동일하게 **거래내역·평단·순위·포트폴리오**에 반영되고, **거래 피드**(예: "🤖 추격이 삼성전자 10주 매수")로 노출해 진행을 생동감 있게 한다.

#### 5.6.3 공정성 · 메달 상호작용 (5.5 연동)

- 봇은 5.6.2의 공정성 원칙(동일 시세·검증)을 따른다.
- **봇의 메달/명예의 전당 자격**: 봇이 1~3등으로 끝났을 때 메달·전당에 어떻게 반영할지 결정 필요. **권장안 — 봇은 메달 비대상**(누적 명예는 사람만), 메달 부여의 "최소 인원"은 **사람 참가자 수** 기준(5.5.1)으로 본다. → 미결.
- **봇 비중이 큰 방의 처리**: 사람 1명 + 봇 다수, 또는 봇만 있는 방에서의 메달·전적 인정 여부. → 미결.

#### 5.6.4 백엔드/데이터 (방향, 상세는 작업 슬러그에서)

- 봇은 **회원(member)의 한 종류(봇 플래그)** 로 표현하고 **참가자로 입장**시켜, 기존 매수/매도·평단·순위·정산 로직을 **그대로 재사용**한다(별도 거래 엔진 없음).
- 봇 매매는 **상주 서버(backend)의 주기 실행**으로 처리한다(시세 적재 직후 1회). 같은 시세 구간에 대한 **중복 실행 방지**가 필요하다.
- 봇 자동 매매는 5.3의 **대결 종료 강제·정산** 흐름과 맞물려, 종료된 방은 더 이상 매매하지 않는다(주기 실행이 종료 처리도 겸할 수 있음).
- 솔로/대결 **거래 코어 공용화(5.3 P2 구조)** 와 정합하도록, 봇 매매도 공용 체결 경로를 탄다.
