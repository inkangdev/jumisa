# 일봉 적재 배치 (closingJob + 과거 데이터 백필) — 2026-06-22 (devlimit)

브랜치: `feat/closing-job`

---

## 배경 / 문제

- 종목 상세 차트에서 KIS API를 사용자 요청마다 온디맨드로 찌름 → 동시 사용자 증가 시 유량(초당 15건) 초과 위험
- `stock_daily`(일봉 테이블)는 스키마는 있으나 `closingJob` 미구현으로 현재 비어있음
- `priceSnapshotJob`은 장중 스냅샷(7일 보관)만 적재하고, 장기 이력은 없음


## 목표

1. **`closingJob`**: 매일 장 마감 후 전 종목 종가를 `stock_daily`에 적재 (진행형 누적)
2. **`historyFillJob`**: 최초 1회 CLI 실행으로 과거 365일치 일봉을 `stock_daily`에 백필
3. **백엔드 차트 엔드포인트**: KIS 온디맨드 → `stock_daily` DB 조회로 전환 (백필 완료 후 즉시 효과)

---

## 설계

### 1. `closingJob` — 매일 15:30 KST 종가 적재

**데이터 소스**: 이미 있는 `KisClient.currentPrice()` (`FHKST01010100`)
- `CurrentPrice` DTO에 종가(`price`), OHLCV, PER/PBR/EPS/BPS/52주 고저가 **모두 포함됨** → 추가 API 불필요

**흐름**:
```
stockRepo.findTradableCodes() → KisClient.currentPrice(code) → StockDailyRow → stock_daily upsert
```

`priceSnapshotJob`과 동일한 chunk reader→processor→writer 패턴, chunk size 100.

**ON CONFLICT DO UPDATE**: 같은 날 재실행해도 덮어쓰기 (멱등).

**스케줄**: `@Scheduled(cron = "0 30 15 * * MON-FRI", zone = "Asia/Seoul")`

**CLI 별칭**: `closing` → `closingJob` (JobCliRunner에 추가)

---

### 2. `historyFillJob` — 과거 365일 백필 (1회성 CLI)

**데이터 소스**: KIS 일별 차트 API `FHKST03010100`
- `batch/KisClient`에 `dailyChart(code, startDate, endDate)` 메서드 추가
- 응답 `output2[]`: `stck_bsop_date`(날짜), `stck_clpr`(종가), `stck_oprc`(시가), `stck_hgpr`(고가), `stck_lwpr`(저가), `acml_vol`(거래량), `acml_tr_pbmn`(거래대금)
- ⚠️ `FHKST03010100`은 PER/PBR/EPS/BPS 없음 → 해당 컬럼은 NULL로 적재

**흐름**:
```
tradableCodes → 종목별 FHKST03010100(기간: today-365 ~ today) → List<StockDailyRow> → stock_daily upsert
```

chunk: 종목 1개 = 1청크 (1종목당 최대 365행 배치 insert)
레이트리밋: `rateLimiter.acquire()` 동일 적용 (~2,400종목 × 1콜, 약 160초)

**스케줄 없음** — CLI 1회 실행 전용:
```bash
docker compose run --rm -e BATCH_SCHEDULER_ENABLED=false batch historyFill
```

---

### 3. 백엔드 차트 엔드포인트 변경

현재: `GET /api/screener/{stockCode}/chart` → `KisChartClient` 온디맨드 호출  
변경 후: `ScreenerRepository.findChartFromDaily(stockCode, startDate)` → `stock_daily` 조회

```sql
SELECT base_date::text, close_price
FROM stock_daily
WHERE stock_code = ? AND base_date >= ?
ORDER BY base_date
```

백필 완료 후 `KisChartClient`와 backend `KisProperties`/`KisChartClient` 빈 제거 가능.

## 미결 논의 사항

1. **장 중 재실행 시 close_price**: `closingJob` 스케줄은 15:30이지만, CLI로 장중 임의 실행하면 현재가가 종가로 들어감. 허용 여부 결정 필요.
2. **백필 PER/PBR 공백**: `historyFillJob`이 채우는 과거 데이터는 PER/PBR NULL. `stock_daily`의 PER/PBR을 차트 기능 외에 다른 곳에서 쓴다면 주의 필요.
3. **Flyway**: 스키마 변경 없음 (`stock_daily`는 `V1__init.sql`에 이미 있음).
