# 대시보드(홈) 탭 추가 — 주요 지수 + 거래량 상위 + 환율(UI)

## 배경
증권 앱 첫 화면처럼 "오늘 시장 한눈에"를 보여주는 새 탭. 디자인 시안
(`docs/design/대시보드-홈.html`)을 먼저 만들고, 거래량·지수는 실데이터로,
환율은 화면(UI)만 추가(데이터 연동 추후)하기로 함.

## 구현 범위
- **거래량 상위**: 기존 `stock_daily` 적재 데이터로 바로 연동 (외부 소스 0).
- **주요 지수(코스피/코스닥/코스피200)**: KIS 국내업종 현재지수 신규 적재.
- **환율**: 프론트 정적 UI만. "연동 예정" 표기. (데이터 연동 추후)

## 변경 사항

### DB
- `V3__market_index.sql` — `market_index_daily` 테이블 신규
  (index_code/index_name/base_date PK, close_index, prev_diff, change_rate,
   OHLC 지수, volume, trade_amount).

### 배치 (지수 적재)
- `dto/KisDtos.kt` — `InquireIndexResponse`, `IndexPrice` 추가.
- `dto/MarketIndexRow.kt` — 행 DTO + `IndexPrice.toIndexRow()` 매퍼.
- `client/KisClient.kt` — `indexPrice(indexCode)` 추가
  (TR **FHPUP02100000**, 시장구분 "U", iscd 0001/1001/2001).
- `repository/IndexRepository.kt` — `insertIndexRows()` upsert.
- `job/MarketIndexJobConfig.kt` — `marketIndexJob` (3개 지수 단일 청크).
- `scheduler/BatchScheduler.kt` — 15:35 MON-FRI 스케줄(종가 잡 직후).
- `JobCliRunner.kt` — `index` → `marketIndexJob` 별칭(+baseDate 파라미터).

### 백엔드 (대시보드 API)
- `dto/DashboardDtos.kt` — `MarketIndex`, `RankItem`, `DashboardResponse`.
- `repository/DashboardRepository.kt` — `findIndices()`(최신 지수),
  `findRanking(rank, limit)`(stock_daily 최신일, vol/up/down).
- `controller/DashboardController.kt` — `GET /api/dashboard?rank=vol|up|down`.
  (SecurityConfig 의 anyRequest().authenticated() 적용 — 스크리너와 동일)

### 프론트
- `api/dashboard.ts` — `fetchDashboard(rank)`.
- `screens/dashboard/DashboardScreen.tsx` — 지수 카드 / 환율 셀(정적) /
  거래량·상승률·하락률 토글 랭킹. 스크리너 행·테마 토큰 재사용.
- `layout/nav.ts` — `home` 탭 추가(🏠, 첫 번째).
- `layout/AppShell.tsx` — 기본 탭 `home`, DashboardScreen 연결.

## 남은 일 / 주의
- **KIS 지수 TR 필드명은 실 키로 검증 필요** — 이 환경에선 KIS 자격증명으로
  실호출 테스트 불가. TR `FHPUP02100000` / output 필드(`bstp_nmix_*`)는
  KIS 문서 기준으로 작성. 운영 키로 1회 `batch index` 수동 실행해 확인 권장.
- 지수 적재 전까지 대시보드 '주요 지수'는 빈 상태("데이터가 아직 없어요").
- 환율은 실데이터 연동(한국수출입은행/한국은행 ECOS 등) 별도 작업.
- 대시보드는 기능정의서(v1.4) 4탭 정의에 없던 신규 화면 → 정식 채택 시
  `기능정의서 vN+1`에 홈 탭 요건 반영 필요.
