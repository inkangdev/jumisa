# 대시보드 지수 5분 실시간 + 화면 자동갱신

작업자: inkangdev / 2026-06-29

## 배경
대시보드 랭킹(거래량/등락)은 시세 스냅샷 기반이라 장중 5분마다 서버에서 갱신되는데,
- **지수(market_index_daily)** 는 하루 1번(15:35)만 적재 → 장중 안 바뀜.
- **화면**은 진입/탭전환 때만 fetch → 5분마다 바뀌어도 자동 반영 안 됨.
둘 다 "5분마다 같이 갱신"되도록 맞춤.

## 수정
### ① 지수 5분 실시간 (`batch/.../scheduler/BatchScheduler.kt`)
- `runMarketIndex` 크론 `0 35 15`(1일1회) → **`0 */5 9-15 * * MON-FRI`**(시세 스냅샷과 동일 주기).
- `market_index_daily` PK=(index_code, base_date), writer 가 `on conflict do update`(upsert)라 **같은 날 행을 5분마다 덮어씀** → 장중 실시간 지수. 마지막 회차(15:55경)가 종가가 됨.
- 지수 3종(코스피/코스닥/코스피200)뿐이라 KIS 호출 부담 없음. baseDate 는 KST(기적용).

### ② 화면 자동갱신 (`frontend/.../dashboard/DashboardScreen.tsx`)
- `useEffect` 에 **60초 폴링**(`setInterval`) 추가 → 안 눌러도 최신값 반영.
- 폴링은 **조용히**(초기 로드만 스피너/에러표시, 폴링 실패는 기존 데이터 유지) → 깜빡임/플리커 방지.
- 언마운트·rank 변경 시 인터벌 정리.

## 검증
- `docker compose build batch web` 컴파일/`tsc` 통과.
- 배포 후: 장중 5분마다 지수값 갱신 + 대시보드 화면이 60초마다 자동 새로고침되는지 확인.

## 비고
- 랭킹·지수 모두 장중 5분 단위 실시간으로 정렬됨(증권사 앱 유사 동작).
