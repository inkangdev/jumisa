# 저평가 스크리너 종목명 검색 추가 + 헤더 정리 — 2026-06-17 (inkangdev)

브랜치: `claude/main-branch-analysis-swtsid` → PR

저평가 스크리너 화면을 보완한다. (1) 헤더에서 종목건수·가이드 문구를 빼 군더더기를 줄이고,
(2) 종목명·코드로 바로 찾는 검색 기능을 추가한다.

## 변경 파일

```
frontend/src/screens/undervalue/UndervalueScreen.tsx
```

## 1. 헤더 정리 (군더더기 제거)

- `{n}개 종목` 카운트 문구 삭제.
- "왼쪽 원형은 저평가 점수(0~100)…" 점수 설명 가이드 문구 삭제.
- 헤더는 **"저평가 스크리너" 제목 + ⚙ 필터 버튼**만 남김.
- 점수 공식 설명은 기존대로 필터 패널 안 안내 박스에 유지(중복 제거).

## 2. 종목명 검색 (클라이언트 즉시 필터)

- 마켓 탭 아래에 🔍 검색 입력창 추가. `종목명·코드 검색` placeholder.
- 입력 즉시 `name`/`stockCode` 부분일치(대소문자 무시) 필터. ✕ 버튼으로 초기화.
- 검색 결과 0건이면 `'<검색어>' 검색 결과가 없습니다` 상태 표시(데이터 미적재 상태와 구분).

## 설계 결정

- **백엔드/API 무변경 — 클라이언트 필터로 처리**: 스크리너는 이미 전 종목(~2,300건)을
  페이징 없이 한 번에 받아오는 구조라, 서버 라운드트립 없이 `items.filter(...)` 로
  입력 즉시 반응시키는 게 가장 간결. `q` 파라미터·쿼리 추가는 불필요.
- 검색은 표시 직전의 `visible` 파생값으로만 적용 → 정렬·필터(PER/PBR/업종)와 독립적으로 합성됨.

## 검증

- `npx tsc --noEmit` 타입체크 통과(에러 0).
- 검색어 입력 시 목록이 즉시 좁혀지고, ✕/공백 시 전체 복귀. 0건 시 안내 상태 노출 확인.

## 메모

- 종목 카드의 관심종목 ☆, 미국 탭은 기존대로 미연동/준비중(이번 범위 아님).
- 적자기업 점수 왜곡(PER/EV NULL → PBR+성장률만으로 상위) 보정은 여전히 별도 후속 후보.
