## 작업 범위

## DB (Supabase SQL 에디터에서 직접 실행)

```sql

-- 소셜 로그인 연동 테이블 (provider 추가 시 행만 늘어남)
create table if not exists social_account (
    id          bigint generated always as identity primary key,
    member_id   bigint not null references member(id) on delete cascade,
    provider    varchar(20)  not null,   -- 'kakao', 'naver', 'google' 등
    provider_id varchar(100) not null,   -- 각 provider 의 사용자 ID
    created_at  timestamptz  not null default now(),
    unique (provider, provider_id)
);
```
 provider 가 늘어도 `social_account`에 행만 추가.

## 카카오 개발자 콘솔 설정

- 카카오 로그인 활성화
- 동의항목: `profile_nickname` 필수 동의
- Redirect URI 등록:
  - `http://localhost:5173/login/oauth2/code/kakao` (로컬)
  - `https://jumisa.onrender.com/login/oauth2/code/kakao` (프로덕션)

## 주요 설계 결정

- **`social_account` 별도 테이블**: `member` 테이블에 `kakao_id` 컬럼 추가 대신 별도 테이블로 분리 → Naver/Google 등 추가 소셜 로그인 확장 대비
- **`client_secret_post` 인증방식**: `client-authentication-method: none` 사용 시 Kakao 토큰 엔드포인트 401. Kakao Client Secret 활성화 앱은 반드시 `client_secret_post`로 설정해야 함
- **redirect_uri**: Vite 프록시가 `changeOrigin: false`로 `Host: localhost:5173` 헤더를 Spring에 전달 → `forward-headers-strategy: framework` 설정 시 Spring이 이를 baseUrl로 인식 → Kakao 콘솔에 5173 포트 URI 등록 필요
- **세션 저장**: `HttpSessionSecurityContextRepository.saveContext()` 직접 호출로 OAuth2 성공 후 세션 생성

## 로그인 플로우

1. 버튼 클릭 → `/oauth2/authorization/kakao` → (Vite/Render 프록시) → Spring
2. Spring이 카카오 인증 페이지로 리다이렉트
3. 카카오 로그인 완료 → `/login/oauth2/code/kakao` → Spring 토큰 교환 (`client_secret_post`)
4. `KakaoOAuth2SuccessHandler`: `social_account` 조회/생성 → 세션 저장 → `/` 리다이렉트
5. React 로드 → `/api/auth/me` → 로그인 상태 확인

## 트러블슈팅

- **`.env` 미인식**: `backend/.env`에 두면 `spring-dotenv`가 찾지 못함. `bootRun` workingDir가 리포 루트이므로 `jumisa/.env`로 이동해야 함
- **Kakao 토큰 교환 401**: `client-authentication-method: none`으로는 Client Secret 활성화 앱에서 무조건 실패. `KakaoTokenClientConfig`의 RestTemplate 인터셉터로 요청 body를 직접 로깅해 파라미터 전달 여부를 확인하고 원인 특정. 최종적으로 올바른 `KAKAO_CLIENT_SECRET` 값 + `client_secret_post` 조합으로 해결
- **`application.yml` 변경 반영 안 됨**: IntelliJ 재시작만으로는 부족. `Build → Rebuild Project` 필수
