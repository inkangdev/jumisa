# 일봉 적재 배치 구현 — 2026-06-23 (devlimit)

브랜치: `feat/closing-job`

---

## 배경

- `stock_daily` 스키마는 `V1__init.sql`에 이미 있으나 적재 배치가 없어 비어있음
- `priceSnapshotJob`이 5분마다 `currentPrice()` (FHKST01010100)를 이미 호출 중
  - 단, `toSnapshot()` 변환에서 PER/PBR/EPS/BPS/52주 고저를 버림 → `stock_price_snapshot` 재활용 불가
  - `currentPrice()` 응답 원본에는 `stock_daily` 전 컬럼이 있으므로 별도 API 추가 없이 재사용 가능
- 종목 상세 차트(현재 KisChartClient 온디맨드)를 백필 완료 후 DB 조회로 전환

---

## 구현 범위

1. **`closingJob`** — 매일 15:30 KST, 전 종목 종가 → `stock_daily` upsert
2. **`historyFillJob`** — 1회성 CLI, 과거 365일 일봉 백필 (FHKST03010100)
3. **`PriceRepository.insertDailyRows()`** — upsert SQL 추가
4. **`BatchScheduler`** — `closingJob` 빈 주입 + 스케줄 등록
5. **`JobCliRunner`** — alias 추가
6. **backend 차트 엔드포인트** — DB 조회로 전환, KisChartClient/KisProperties 제거

Flyway 변경 없음 — `stock_daily`는 `V1__init.sql`에 존재.

```kotlin
/** 국내주식 일별 차트 (TR: FHKST03010100). 기간: yyyyMMdd 형식. 실패 시 빈 리스트. */
fun dailyChart(stockCode: String, startDate: String, endDate: String): List<DailyChartItem> {
    rateLimiter.acquire()
    val body = rest.get()
        .uri {
            it.path("/uapi/domestic-stock/v1/quotations/inquire-daily-itemchartprice")
                .queryParam("fid_cond_mrkt_div_code", "J")
                .queryParam("fid_input_iscd", stockCode)
                .queryParam("fid_input_date_1", startDate)
                .queryParam("fid_input_date_2", endDate)
                .queryParam("fid_period_div_code", "D")
                .queryParam("fid_org_adj_prc", "0")
                .build()
        }
        .header("authorization", "Bearer ${accessToken()}")
        .header("appkey", props.appKey)
        .header("appsecret", props.appSecret)
        .header("tr_id", "FHKST03010100")
        .retrieve().body(String::class.java) ?: return emptyList()

    return try {
        val resp = objectMapper.readValue(body, DailyChartResponse::class.java)
        if (resp.rtCd != "0") {
            log.warn("[{}] 일별 차트 조회 실패 rt_cd={} {}", stockCode, resp.rtCd, resp.msg)
            emptyList()
        } else {
            resp.output2 ?: emptyList()
        }
    } catch (e: Exception) {
        log.warn("[{}] 일별 차트 파싱 실패: {}", stockCode, e.message)
        emptyList()
    }
}
```

## 3. `PriceRepository.kt` — `insertDailyRows()` 추가

```kotlin
fun insertDailyRows(rows: List<StockDailyRow>) {
    if (rows.isEmpty()) return
    val sql = """
        insert into stock_daily
          (stock_code, base_date, close_price, open_price, high_price, low_price,
           volume, trade_amount, market_cap_eokwon, per, pbr, eps, bps, w52_high, w52_low)
        values (?,?,?,?,?,?,?,?,?,?,?,?,?,?,?)
        on conflict (stock_code, base_date) do update set
          close_price       = excluded.close_price,
          open_price        = excluded.open_price,
          high_price        = excluded.high_price,
          low_price         = excluded.low_price,
          volume            = excluded.volume,
          trade_amount      = excluded.trade_amount,
          market_cap_eokwon = excluded.market_cap_eokwon,
          per               = excluded.per,
          pbr               = excluded.pbr,
          eps               = excluded.eps,
          bps               = excluded.bps,
          w52_high          = excluded.w52_high,
          w52_low           = excluded.w52_low
    """.trimIndent()

    jdbc.batchUpdate(sql, object : BatchPreparedStatementSetter {
        override fun setValues(ps: PreparedStatement, i: Int) {
            val r = rows[i]
            ps.setString(1, r.code)
            ps.setObject(2, r.baseDate)
            ps.setInt(3, r.closePrice)
            ps.setObject(4, r.openPrice)
            ps.setObject(5, r.highPrice)
            ps.setObject(6, r.lowPrice)
            ps.setObject(7, r.volume)
            ps.setObject(8, r.tradeAmount)
            ps.setObject(9, r.marketCapEok)
            ps.setObject(10, r.per)
            ps.setObject(11, r.pbr)
            ps.setObject(12, r.eps)
            ps.setObject(13, r.bps)
            ps.setObject(14, r.w52High)
            ps.setObject(15, r.w52Low)
        }
        override fun getBatchSize() = rows.size
    })
}
```

## 4. `ClosingJobConfig.kt` — 신규

`batch/job/ClosingJobConfig.kt`

```kotlin
@Configuration
class ClosingJobConfig(
    private val jobRepository: JobRepository,
    private val txManager: PlatformTransactionManager,
) {
    @Bean
    fun closingJob(closingStep: Step): Job =
        JobBuilder("closingJob", jobRepository).start(closingStep).build()

    @Bean
    fun closingStep(
        closingCodeReader: ItemReader<String>,
        closingProcessor: ItemProcessor<String, StockDailyRow>,
        closingWriter: ItemWriter<StockDailyRow>,
    ): Step =
        StepBuilder("closingStep", jobRepository)
            .chunk<String, StockDailyRow>(100, txManager)
            .reader(closingCodeReader)
            .processor(closingProcessor)
            .writer(closingWriter)
            .build()

    @Bean
    @StepScope
    fun closingCodeReader(stockRepo: StockRepository): ItemReader<String> =
        ListItemReader(stockRepo.findTradableCodes())

    @Bean
    @StepScope
    fun closingProcessor(
        kis: KisClient,
        @Value("#{jobParameters['baseDate']}") baseDateStr: String,
    ): ItemProcessor<String, StockDailyRow> {
        val date = LocalDate.parse(baseDateStr)
        return ItemProcessor { code -> kis.currentPrice(code)?.toDailyRow(code, date) }
    }

    @Bean
    fun closingWriter(priceRepo: PriceRepository): ItemWriter<StockDailyRow> =
        ItemWriter { chunk -> priceRepo.insertDailyRows(chunk.items) }
}
```

## 5. `HistoryFillJobConfig.kt` — 신규

`batch/job/HistoryFillJobConfig.kt`

종목별로 365일 범위 1콜 → 최대 365행 배치 insert. Tasklet 방식(전 종목 순회).

```kotlin
@Configuration
class HistoryFillJobConfig(
    private val jobRepository: JobRepository,
    private val txManager: PlatformTransactionManager,
) {
    @Bean
    fun historyFillJob(historyFillStep: Step): Job =
        JobBuilder("historyFillJob", jobRepository).start(historyFillStep).build()

    @Bean
    fun historyFillStep(historyFillTasklet: Tasklet): Step =
        StepBuilder("historyFillStep", jobRepository)
            .tasklet(historyFillTasklet, txManager)
            .build()

    @Bean
    fun historyFillTasklet(
        stockRepo: StockRepository,
        kis: KisClient,
        priceRepo: PriceRepository,
    ): Tasklet = Tasklet { _, _ ->
        val fmt = DateTimeFormatter.ofPattern("yyyyMMdd")
        val end = LocalDate.now().format(fmt)
        val start = LocalDate.now().minusDays(365).format(fmt)

        val codes = stockRepo.findTradableCodes()
        val log = LoggerFactory.getLogger(javaClass)
        for (code in codes) {
            val rows = kis.dailyChart(code, start, end).mapNotNull { it.toDailyRow(code) }
            if (rows.isNotEmpty()) priceRepo.insertDailyRows(rows)
            log.debug("[{}] 일봉 {}건 적재", code, rows.size)
        }
        RepeatStatus.FINISHED
    }
}
```

## 실행 순서

```bash
# 1. 과거 365일 백필 (최초 1회, 약 2~3분 소요)
docker compose run --rm -e BATCH_SCHEDULER_ENABLED=false batch historyFill

# 2. 당일 종가 수동 적재 (백필 후 바로 오늘 것도 넣고 싶을 때)
docker compose run --rm -e BATCH_SCHEDULER_ENABLED=false batch closing

# 이후 상주 스케줄러가 매일 15:30 자동 실행
```

## 주의

- `historyFillJob`의 `FHKST03010100`은 PER/PBR/EPS/BPS 없음 → NULL 적재. 차트 표시에는 문제 없음.
- `closingJob` CLI를 장중에 수동 실행하면 현재가가 종가로 들어감. ON CONFLICT DO UPDATE라 15:30 자동 실행 시 덮어씌워짐.
- KIS 레이트리밋(초당 15건) — `rateLimiter.acquire()`가 이미 적용돼 있으므로 별도 처리 불필요.
