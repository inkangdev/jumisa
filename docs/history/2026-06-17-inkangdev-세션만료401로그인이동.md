# 2026-06-17 · 세션 만료(401) 시 로그인 화면으로 자동 이동

브랜치: `feature/session-expiry-redirect` (main 기준)

세션이 만료된 뒤 인증이 필요한 API 가 401 을 반환하면, 화면 전환 없이 에러만 나던 것을 **로그인 화면으로 자동 이동 + 안내 문구** 표시로 바꾼다.

## 세션 유지 시간 (확인)

- `backend/.../application.yml` 에 `server.servlet.session.timeout` 설정 없음 → **Spring Boot 기본값 30분** (마지막 요청 기준 idle 30분 후 만료).
- 이번 작업에서 시간 자체는 변경하지 않음(요청 범위 밖).

## 한 일 (프론트 전용)

- `api/session.ts` (신규) — 401 전역 신호. `onUnauthorized(cb)` 등록 / `notifyUnauthorized()` 발화.
  - 로그인/세션확인(`me`) 의 401 은 만료가 아니라 정상 흐름 → 여기서 다루지 않음.
- `api/battle.ts`, `api/screener.ts` — 인증 필요한 `/api` 호출이 401 이면 `notifyUnauthorized()` 호출 후 기존 에러 반환.
- `api/auth.ts` — `postJson` 에 `notifyOn401` 플래그 추가. **아바타·닉네임 변경에만 true**(로그인/회원가입 401 은 자격증명 실패라 제외).
- `App.tsx` — `onUnauthorized` 핸들러 등록: `setUser(null)` + 안내 문구("세션이 만료되었습니다. 다시 로그인해 주세요.") + 로그인 화면 전환.
- `/ai/ask` 는 별도 FastAPI(세션 무관)라 손대지 않음.

## 검증

- 프론트 `tsc --noEmit` 통과.
- 수동(예정): 로그인 → 30분 이상 idle(또는 백엔드 세션 만료) → 스크리너/배틀/내정보 조작 시 401 → 로그인 화면 + 안내 문구.

## 비고

- 세션 시간을 바꾸려면 `application.yml` 에 `server.servlet.session.timeout: 30m` 형태로 명시(미설정 시 30분).
