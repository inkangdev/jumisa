# 배치 수동 실행 워크플로 추가 — 2026-06-24 (devlimit)

브랜치: `main` (직접 커밋)

---

## 배경 / 문제

- `historyFillJob`(365일 일봉 백필)을 최초 1회 실행해야 `stock_daily`가 채워지고 차트가 동작함
- 운영 서버(self-hosted runner가 있는 Windows PC)에 직접 접속해서 `docker compose run ...` 명령을 치는 건 번거롭고 접근 권한이 필요함
- 기존 `deploy.yml`은 `main` push 시 자동 배포만 담당 — 배치 수동 실행 방법이 없었음

## 해결

GitHub Actions `workflow_dispatch` 트리거를 이용한 수동 실행 워크플로 추가.
self-hosted runner(운영 PC)가 받아서 운영 DB에 직접 적재.

---

## 추가 파일

`.github/workflows/batch-run.yml`

---

## 사용 방법

### 최초 1회 — 365일 일봉 백필 (historyFill)

1. GitHub 웹사이트 접속
2. 레포 → **Actions** 탭
3. 왼쪽 목록에서 **batch run** 클릭
4. 오른쪽 상단 **Run workflow** 버튼 클릭
5. `job` 드롭다운에서 **historyFill** 선택
6. **Run workflow** 클릭

약 10~15분 소요 (2,400 종목 × 4구간 API 호출, 레이트리밋 적용).
완료되면 `stock_daily`에 365일치 일봉이 적재되고, 종목 상세 차트가 동작함.

### 이후 — 매일 자동

배포된 batch 컨테이너가 **평일 15:30 KST** 에 `closingJob`을 자동 실행 → 당일 종가 1일치 추가 적재.
별도 조작 불필요.

### 다른 배치 수동 실행

같은 방법으로 `job` 드롭다운에서 선택 가능:

| job | 설명 | 비고 |
|-----|------|------|
| `historyFill` | 365일 일봉 백필 | 최초 1회만 |
| `closing` | 당일 종가 1일치 적재 | 15:30 자동 실행 안 기다릴 때 |
| `master` | 종목 마스터 갱신 | 평일 08:00 자동 실행 |
| `finance` | 재무지표 갱신 | 일요일 06:00 자동 실행 |
| `undervalue` | 저평가 점수 산출 | 매시 정각 자동 실행 |

---

## 주의

- `workflow_dispatch`는 **버튼을 눌러야만** 실행됨 — push 시 자동 실행 아님
- `historyFill`은 ON CONFLICT DO UPDATE라 여러 번 실행해도 데이터 중복 없음 (덮어쓰기)
- 실행 중 로그는 Actions 탭 → batch run → 해당 실행 클릭 → `배치 잡 실행` 스텝에서 확인 가능
- `.env` 파일은 러너 PC의 `C:\jumisa-secrets\.env` (또는 `JUMISA_ENV_FILE` 환경변수 경로)에서 자동 주입됨
