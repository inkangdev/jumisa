## 차트 데이터 방식: Option B — KIS 온디맨드 API

`stock_daily`(closingJob 미구현, 빈 테이블)와 `stock_price_snapshot`(7일 보관)으로는 1M+ 차트 제공 불가.  
→ 차트 조회 요청 시 **KIS `FHKST03010100` API를 온디맨드 호출**로 결정.

KIS API:
- 엔드포인트: `/uapi/domestic-stock/v1/quotations/inquire-daily-itemchartprice`
- headers: `authorization: Bearer {token}`, `appkey`, `appsecret`, `tr_id: FHKST03010100`
- params: `fid_cond_mrkt_div_code=J`, `fid_input_iscd={code}`, `fid_input_date_1/2`, `fid_period_div_code=D`, `fid_org_adj_prc=0`
- KIS 키 미설정 시 빈 배열 반환 (에러 없이 graceful 처리)

## 1. 백엔드

### 변경 파일

| 파일 | 내용 |
|------|------|
| `AppApplication.kt` | `@ConfigurationPropertiesScan` 추가 |
| `application.yml` | `kis.env / app-key / app-secret` 섹션 추가 |
| `config/KisProperties.kt` | **신규** — `@ConfigurationProperties("kis")` 바인딩 |
| `client/KisChartClient.kt` | **신규** — KIS 토큰 인메모리 캐시 + `fetchDailyChart()` |
| `dto/ScreenerDtos.kt` | `StockDetail`, `ChartPoint`, `StockChartResponse` 추가 |
| `repository/ScreenerRepository.kt` | `findDetail(stockCode)` 추가 |
| `controller/ScreenerController.kt` | `GET /{stockCode}`, `GET /{stockCode}/chart?period=` 추가 |

### 신규 엔드포인트

```
GET /api/screener/{stockCode}
→ StockDetail (undervalue_score + stock + stock_price_snapshot 최신 조인)

GET /api/screener/{stockCode}/chart?period=1M|3M|6M|1Y
→ StockChartResponse (KIS 온디맨드 조회)
```

period → days 매핑: 1M=30, 3M=90, 6M=180, 1Y=365
