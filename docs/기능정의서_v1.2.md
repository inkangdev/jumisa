## 5. 모의투자 대결 (멀티플레이) — v1.2 보완판

### 5.1 기존 요건 (v1.1 유지)

- **로비**: 참가 가능한 방 / 진행 중 대결 목록, 새 방 만들기
- **방 생성**: 이름, 기간(3/7/14/30일), 시작 포인트(50만/100만/300만P), 최대 인원(2~20명), 거래 시장(국내+미국/국내/미국)
- **대기방**: 초대 코드 공유, 참가자 목록·준비 상태, 방장 시작, 시작 시 시작 포인트 동시 지급(시작 후 충전 불가)
- **진행**: 내 순위 카드(순위/수익률/총자산)·D-day, 순위 탭, 내 투자 탭, 거래 탭
- **순위 기준**: 종료 시점 **총자산(현금+주식평가액)** 의 수익률

### 5.2 1차 구현 현황 (인지)

방 생성·입장·시작·매수/매도·순위·포트폴리오까지 동작하는 1차 슬라이스가 머지됨(국내 종목, 1시간 적재가 기준). **단, 아래 "거래 시스템 4대 요건"이 미비하여 정확성·공정성·완결성 보강이 필요하다.**

### 5.3 보완 필요 항목 (우선순위별)

> 거래 시스템의 4대 기둥 = **정합성 / 가격 / 정산 / 회계**. 여기에 인가·성능·구조를 더한다.

#### P0 — 정합성·안전 (먼저 잡아야 신뢰 가능)

- **동시 거래 안전성**: 한 참가자의 현금·보유 수량이 동시 요청에 서로 덮어써지지 않도록 보장. (잔고 초과 매수, 음수 수량 같은 "돈이 새는" 상태 방지)
- **거래 멱등성**: 더블클릭·재전송으로 동일 거래가 중복 체결되지 않도록.
- **대결 종료 강제**: 종료 시각(ends_at) 이후 거래 차단. (현재는 마감 후에도 무기한 거래 가능)
- **결과 확정(정산)**: 종료 시 최종 순위·총자산을 **고정(freeze)** 하고 상태를 '종료'로 전환. 종료 후 시세가 바뀌어도 결과·우승자가 변하지 않도록. (대결을 로컬 배치 스케줄러에 의존시키면 PC가 꺼졌을 때 정산이 안 되므로, **조회/접근 시점에 정산하는 방식**을 고려)
- **조회 권한(인가)**: 비참가자가 임의 방의 상세·**초대코드**를 보지 못하도록. (현재 누구나 조회 → 초대코드 누출)
- **시세 부재 처리**: 현재가가 없을 때 평가액을 평단으로 **조용히 대체하지 말 것**. 지금은 수익이 0%로 숨겨져 순위가 왜곡됨 → 명시적으로 "가격 없음" 표시.
- **초대코드 충돌 처리**: 코드 중복 생성 시 재시도(현재 충돌하면 방 생성 실패).

#### P1 — 거래 현실성·회계

- **체결가 기준 확정(선결 결정)**: 현재 "1시간 적재가"는 한 시간 동안 가격이 안 움직여 장중 변동·공정성이 없음. 실시간 / 더 잦은 적재 / 체결 시점 조회 중 택1.
- **시세 신뢰성**: 시세 수집이 멈추면 거래·평가가 불가능/왜곡됨. 대결은 라이브 기능이므로 시세 공급의 신뢰성 확보 필요.
- **종목별 최신가 캐시**: 평가·순위 때마다 시세 이력을 반복 조회하지 않도록 최신가를 빠르게 얻는 구조.
- **거래 원장 보강**: 거래마다 수수료/세금·체결 후 잔고·**실현손익**을 기록 → 포지션·현금과 대사(검증) 가능, 실현손익 표시 가능.
- **금액 정밀도**: 평단·평가의 정수 절삭으로 인한 손익 오차 제거.
- **시작 조건**: 최소 인원·준비 상태 충족 시에만 시작(현재 방장 혼자 시작 가능, 준비 상태는 표시만 있고 실제 없음).

#### P2 — 스펙 완성·확장

- **거래시장 설정 실효화**: 현재 kr/us/both 설정이 실제 거래에 반영되지 않음. 미국 종목·환율 데이터가 없어 'us'/'both' 방은 사실상 거래 불가.
- **미국 종목 + 환율**: 미국 시세·재무 + **체결 시점 실시간 환율(스펙 확정)** 을 매매·평가액에 적용.
- **결과 영속화**: 종료된 대결의 최종 순위·실현손익·우승자를 보관(히스토리·재조회).
- **순위·평가 성능**: 참가자·보유 수에 비례하는 반복 조회를 단일 집계로. 거래 종목 목록이 일부만 노출되는 것도 정리.
- **거래 코어 공용화(구조)**: 솔로 모의투자(C)와 대결(D)이 **가격 조회·체결·평단·현금·환율·회계** 로직을 공유하도록 공용 모듈로 추출. (대결이 자체 구현 중 → 솔로 구현 시 중복·표류 방지)

#### P3 — 부가

- 거래 화면 실시간 가격 갱신(폴링 → 효율적 방식)
- 대결 진행 상태 영속(새로고침·탭 전환 대응)
- 가격 알림 / 랭킹 히스토리 / 봇 참가자(선택)

### 5.4 선결 결정 (코드 전 합의 필요)

- **대결 시세 갱신 주기** — 실시간 vs 1시간 적재가

- **수수료·거래세 적용 여부** — 현실성 vs 단순함
