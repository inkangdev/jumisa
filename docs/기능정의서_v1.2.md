# Jumisa 기능 정의서 v1.2

> 기준 시안: `docs/design/remixed-13412f58.tsx`
> v1.2는 v1.1 전체를 유지하고, **모의투자 대결(5장)** 에 1차 구현 리뷰 기반 **보완 항목**을 추가한다.

## 0. 화면 구성 (하단 네비게이션)

시안 기준 4개 탭: **스크리너 / 대결 / 관심종목 / 더보기**

## 1. 저평가 주식 스크리너

- 국내/미국 탭 구분 (**미국 포함 확정**)
- 섹터/업종 필터, PER·PBR·EV/EBITDA 상한 슬라이더 필터
- 정렬: 점수순 / PER순 / PBR순
- 저평가 점수 산출 및 랭킹 (점수·색상 뱃지 표시)
- PER/PBR/EV/EBITDA 등 주식 용어 → 입문자용 쉬운 용어 병행 표기
  (예: PER "주가÷순이익", PBR "주가÷순자산", EV/EBITDA "기업가치÷영업이익")
- 종목 → 관심종목 ☆ 토글, 종목 클릭 시 거래 화면 진입

### 저평가 점수 공식 (확정)

`점수 = PER 30% + PBR 30% + EV/EBITDA 25% + 성장률 15%`

### 데이터 소스

- 국내 주가 (1시간 단위) → 한국투자증권 Open API
- 국내 재무지표 → 금융위원회 공공데이터포털
- 미국 주식 (시세·재무) → FMP API

### 뉴스 / AI 분석

- 네이버 금융 RSS로 종목별 뉴스 수집
- Claude API로 뉴스 감성 분석 및 종목 현황 요약 제공
- 미국 종목 뉴스 → FMP 뉴스 엔드포인트
- ※ 시안에 화면 미정의 — 화면 설계부터 필요

## 2. 로그인 / 회원

### 로그인

- 아이디(닉네임)/비밀번호 + 간편 소셜 로그인 (카카오 / 네이버)
- 가입 시 아바타(이모지) 선택
- 가입 즉시 초기 포인트 **1,000,000P** 지급
- 비밀번호 변경 / 회원 탈퇴

### 더보기 (회원 정보)

- 프로필: 별명 / 아바타 / 이메일 / 보유 포인트
- 포인트 충전, 로그아웃
- 공지사항 / FAQ
- 모의투자 포인트 / 보유 종목 / 수익률 통계
- 보유 주식 관리 (수동 입력, 종목 코드 입력 시 현재가 자동 연동)

## 3. 공통

- 관심 종목 등록 및 가격 변동 알림 (※ 알림은 시안 미정의)
- 저평가 랭킹 히스토리 (순위 추이 확인, ※ 시안 미정의)
- 공지사항 / FAQ 페이지

## 4. 모의투자 (솔로)

### 포인트

- 가입 시 초기 포인트 **1,000,000P** 지급 (1포인트 = 1원)
- 포인트 충전 가능 (재충전 정책 상세 확정 필요)

### 매수 / 매도

- 기준가 → 1시간 단위 적재 주가 기준
- 시장가 즉시 체결 방식
- 평균단가·손익률 계산, 거래내역 기록
- 해외 종목 → 매수 시점 **실시간 환율 연동** (체결 시점 환율 적용)

## 5. 모의투자 대결 (멀티플레이)

> 시안의 핵심 기능. 같은 시작 포인트로 출발해 기간 내 수익률로 순위를 겨룬다.

### 5.1 요건

- **대결 로비**: 참가 가능한 방 목록 / 진행 중인 대결 목록, 새 대결방 만들기
- **방 생성**: 방 이름, 대결 기간(3 / 7 / 14 / 30일), 시작 포인트(50만 / 100만 / 300만P), 최대 인원(2~20명), 거래 가능 시장(국내+미국 / 국내 / 미국)
- **대기방**: 초대 코드 공유, 참가자 목록/준비 상태, 방장이 대결 시작. 시작 시 모든 참가자에게 시작 포인트 동시 지급, 시작 후 추가 충전 불가
- **대결 진행**: 내 순위 카드(순위/수익률/총자산)·D-day, 순위 탭(랭킹+수익률 막대그래프), 내 투자 탭(현금/주식평가액/보유종목 손익), 거래 탭(시장 필터 국내/미국)
- **순위 기준**: 종료 시점 **총자산(현금 + 주식평가액)** 의 수익률

### 5.2 1차 구현 현황 (인지)

방 생성·입장·시작·매수/매도·순위·포트폴리오까지 동작하는 1차 슬라이스가 머지됨(국내 종목, 1시간 적재가 기준). **단, 거래 시스템으로서의 정확성·공정성·완결성 요건이 미비하여 아래 보완이 필요하다.**

### 5.3 보완 필요 항목 (우선순위별)

> 거래 시스템의 4대 기둥 = **정합성 / 가격 / 정산 / 회계** + 인가·성능·구조.

**P0 — 정합성·안전**

- **동시 거래 안전성**: 한 참가자의 현금·보유 수량이 동시 요청에 덮어써지지 않도록(잔고 초과 매수·음수 수량 방지).
- **거래 멱등성**: 더블클릭·재전송 중복 체결 방지.
- **대결 종료 강제**: 종료 시각 이후 거래 차단(현재 마감 후에도 무기한 거래 가능).
- **결과 확정(정산)**: 종료 시 최종 순위·총자산을 고정(freeze)하고 '종료'로 전환. 종료 후 시세 변동이 결과·우승자를 바꾸지 않도록. (로컬 배치 스케줄러 의존 시 PC 꺼지면 정산 누락 → **조회 시점 정산** 방식 고려)
- **조회 권한(인가)**: 비참가자가 임의 방의 상세·**초대코드**를 보지 못하도록.
- **시세 부재 처리**: 현재가가 없을 때 평가액을 평단으로 조용히 대체하지 말 것(수익이 0%로 숨겨져 순위 왜곡). 명시적으로 "가격 없음" 표시.
- **초대코드 충돌 처리**: 중복 생성 시 재시도.

**P1 — 거래 현실성·회계**

- **체결가 기준 확정(선결)**: "1시간 적재가"는 한 시간 동안 가격이 안 움직여 장중 변동·공정성이 없음. 실시간/더 잦은 적재/체결 시점 조회 중 택1.
- **시세 신뢰성**: 시세 수집이 멈추면 거래·평가 불가/왜곡. 라이브 기능이므로 시세 공급 신뢰성 확보.
- **종목별 최신가 캐시**: 평가·순위 때마다 시세 이력을 반복 조회하지 않도록.
- **거래 원장 보강**: 거래마다 수수료/세금·체결 후 잔고·**실현손익** 기록 → 대사 가능, 실현손익 표시.
- **금액 정밀도**: 평단·평가의 정수 절삭 오차 제거.
- **시작 조건**: 최소 인원·준비 상태 충족 시에만 시작(현재 방장 혼자 시작 가능, 준비 상태는 표시만).

**P2 — 스펙 완성·확장**

- **거래시장 설정 실효화**: 현재 kr/us/both 설정이 거래에 반영되지 않음. 미국 종목·환율 없이 'us'/'both' 방은 거래 불가.
- **미국 종목 + 환율**: 미국 시세·재무 + 체결 시점 실시간 환율을 매매·평가액에 적용.
- **결과 영속화**: 종료된 대결의 최종 순위·실현손익·우승자 보관.
- **순위·평가 성능**: 참가자·보유 수에 비례하는 반복 조회를 단일 집계로. 거래 종목 일부만 노출되는 문제 정리.
- **거래 코어 공용화(구조)**: 솔로 모의투자(4장)와 대결(5장)이 가격 조회·체결·평단·현금·환율·회계 로직을 공유하도록 공용화 → 중복·표류 방지.

**P3 — 부가**

- 거래 화면 실시간 가격 갱신 / 진행 상태 영속(새로고침 대응) / 가격 알림·랭킹 히스토리·봇(선택).

### 5.4 선결 결정 (코드 전 합의)

- 대결 시세 갱신 주기 (실시간 vs 적재가)
- 시세 수집 신뢰성 방식
- 수수료·거래세 적용 여부
